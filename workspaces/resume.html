Fleet Engine: AI GTM Enrichment Pipeline — Architect & Builder

2025–Present | Independent systems build | portfolio.blueskygtm.engineer
  • Designed and built a three-agent AI pipeline — Ahab (discovery), Nemo (enrichment), Neptune (outreach synthesis) — operating sequentially from public job listing signals through CRM-ready payload delivery to Clay and HubSpot.
  • V1 (Fathom): Ran on a self-hosted n8n + Postgres + Docker stack on GCP. Completed 1,500 enrichment cycles across six campaigns, generating 500+ actionable leads. Documented 18 environmental failure modes across scraping, routing, and output QA — none touched the agent prompts.
  • V2 (Cortex): Rebuilt the infrastructure from scratch. Replaced Docker/Postgres/n8n with Google Cloud Storage file handoff (each agent writes a typed JSON file, the next stage gates on it), eliminating every V1 failure mode by removing the failure domain entirely.
  • Deployed Python ADK (Google Agent Development Kit) orchestration layer on Google Agent Engine — LlmAgent definitions, RagTool integrations, and a Pipeline_Orchestrator as the ADK entry point. Node.js path retained for local iteration; ADK path for hosted execution.
  • Provisioned and configured three Vertex AI RAG corpora (us-central1, Serverless mode) — Ahab deduplication vault, Nemo friction calibration store, Neptune outreach variation store — compounding across every run.
  • Built a Python Pydantic validation layer: 8 production control patterns covering schema enforcement, state machine transitions, LLM output cleanup (citation stripping, grounding URL sanitization), and GCS IPC contract validation.
  • Agent prompts written as version-controlled instruction sets using the Eugene Schwartz direct-response methodology — transferred verbatim from V1 to V2, validated across 1,500+ cycles. Neptune output governed via responseSchema enforcement at the Vertex AI API level.
  • Orchestrated entirely through Claude Code as Admiral — no Express server, no polling middleware, no connection pools. GCS files are the only shared state between pipeline stages.
